# 通用安全重构搬迁技能（Copy-First）

## 适用场景
- 将 `src/shared/*`、`src/common/*`、`src/global/*` 中任意文件按业务模块下沉。
- 将全局文件按模块拆分（如 `components`、`hooks`、`utils`、`constants`、`services`、`types`、`styles`、`api`、`state`）。
- 需要在重构期间保持可回滚：先复制，暂不删除旧文件。
- “架构调整”，禁止“逻辑改写”。

## 非目标
- 不做函数逻辑重写或行为变更。
- 不做 API 语义变更。
- 不在未获明确许可时删除源文件。

## 输入约定（开始前确认）
1. 源文件/目录列表。
2. 目标映射规则（每个符号/文件片段归属到哪个模块文件）。
3. 是否开启“只复制不删除”（默认开启）。
4. 是否执行构建验证（默认执行 `npm run build`）。
5. 如果冲突：采用“保持原逻辑优先，命名最小改动”的策略。

## 执行流程
1. 建立迁移映射表。
2. 在目标模块中创建缺失文件/目录（按目标结构最小创建）。
3. 复制符号或文件内容到目标文件，保持实现与对外行为不变。
4. 更新相关 `import`/`export` 路径，确保引用指向新位置。
5. 处理 barrel 文件（如 `index.ts`）导出，避免出现悬空导入。
6. 运行构建检查：`npm run build`。
7. 若构建失败，仅修复路径、导出、可见性、命名冲突等迁移相关问题，不引入逻辑改动。
8. 输出迁移结果清单（新增文件、修改文件、未处理项）。
9. 未得到“可删除旧文件”的明确指令前，不删除源文件中的原定义。

## 分支决策规则
- 目标文件不存在：创建文件并补最小必要导出。
- 同名符号冲突：
  1. 优先保持既有公共 API 不变；
  2. 在目标文件做最小重命名（如后缀化）并同步修正引用；
  3. 记录冲突处理原因。
- 不确定归属模块：暂停该符号迁移并列入“待确认项”，先完成可确定部分。

## 质量门禁（完成标准）
- 逻辑等价：迁移前后行为与语义不变。
- 路径正确：不存在无效 import 或循环导入恶化。
- 可构建：`npm run build` 通过。
- 可追踪：提供文件级变更清单与待确认项。
- 可回滚：在 copy-first 模式下，旧文件内容仍保留。

## 输出模板
- 迁移概览：源 -> 目标映射摘要
- 文件变更：新增/修改列表
- 校验结果：build 成功或失败摘要
- 风险与待确认：冲突、归属不明确、潜在重复定义
- 下一步建议：在确认无回归后统一删除旧定义

## 推荐触发词
- “按模块重构 shared/common 文件”
- “将全局文件下沉到业务模块”
- “只复制不删除地重构”
- “迁移任意文件并修复 import/export 路径”
